Техническое задание на разработку фронтенда (Angular)
Платформа агрегации внешних курсов для сотрудников РЖД
1. Общие сведения
1.1 Наименование проекта
Платформа организации обучения "РЖД-Академия" - система управления выбором и прохождением внешних курсов от агрегаторов

1.2 Цель проекта
Создание платформы для выбора, утверждения и сопровождения внешних курсов от различных образовательных агрегаторов. Система не предоставляет контент, а управляет процессом обучения на внешних платформах.

1.3 Основные отличия от стандартной LMS:
Нет внутреннего контента - курсы проводятся на внешних платформах

Процесс выбора - сотрудник выбирает 3 курса из предложенных по специальности

Утверждение администратором - обязательное согласование выбора

Интеграция с внешними системами - ссылки на Яндекс.Телемост и другие платформы

Загрузка результатов - сотрудники загружают сертификаты после прохождения

2. Технологический стек
2.1 Основные технологии
Фреймворк: Angular 16+

Язык: TypeScript

UI-библиотека: Angular Material + PrimeNG (для расширенных компонентов)

Стилизация: SCSS с кастомной темой РЖД

Состояние: NgRx (для сложных состояний выбора курсов)

HTTP-клиент: Angular HttpClient с interceptors

WebSocket: Socket.IO Client для чата

Календарь: FullCalendar Angular

Файлы: ngx-uploader для загрузки сертификатов

PDF-просмотр: ng2-pdf-viewer

Уведомления в реальном времени: Angular Service Worker + Web Push

typescript
interface ExternalCourse {
  id: string;
  title: string;
  description: string;
  provider: string;           // Агрегатор (Coursera, Stepik, Яндекс.Практикум и т.д.)
  externalLink: string;       // Ссылка на страницу курса у агрегатора
  dates: {
    start: Date;
    end: Date;
    sessions: Session[];      // Расписание занятий
  };
  format: 'online' | 'offline' | 'hybrid';
  location?: string;          // Для оффлайн курсов
  price: number;
  currency: 'RUB' | 'USD' | 'EUR';
  capacity: number;           // Максимальное количество участников
  enrolled: number;           // Уже записано
  requirements: string[];     // Требования к участникам
  specializations: string[];  // Специальности, для которых подходит
}

5. Процесс обучения (User Flow)
5.1 Полный цикл для сотрудника:
text
1. Уведомление о начале периода выбора курсов
2. Выбор 3 курсов из каталога → Отправка на утверждение
3. Ожидание решения администратора (1-3 дня)
4. Получение уведомления об утверждении/отклонении
5. За 3 дня до курса: подтверждение участия
6. Получение инструкций и ссылки на внешнюю платформу
7. Участие в курсе на внешней платформе
8. Общение в групповом чате с коллегами
9. После завершения: загрузка сертификата
10. Сертификат проверяется администратором
11. Курс отмечается как завершенный
5.2 Процесс для администратора:
text
1. Настройка периода выбора (даты, бюджет, курсы)
2. Мониторинг выбора сотрудников
3. Утверждение/отклонение заявок
4. Контроль подтверждения участия
5. Мониторинг посещаемости (интеграция с агрегаторами)
6. Проверка загруженных сертификатов
7. Формирование отчетов по завершению периода
